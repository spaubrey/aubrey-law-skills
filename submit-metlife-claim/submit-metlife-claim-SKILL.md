# Submit MetLife Claim — Portal Submission

Submits each line item from a MetLife claim doc as a separate claim entry on the
MetLife attorney portal at `attorneys.legalplans.com`. One portal submission per
document line item.

> **Prerequisite**: Claude in Chrome must be connected. If not available, stop and
> tell Scott to enable it in the Tools menu before proceeding.

---

## How MetLife Portal Claims Work

Each document in the claim maps to **one portal submission**. A matter with 14 line
items requires 14 separate submissions. Each submission uses:
- The **MetLife service type** (not the document name — see mapping below)
- The **individual line item fee** (not the section total)
- The **comment** from the claim doc

### Service type mapping

| Document name in claim doc | Portal service type |
|----------------------------|---------------------|
| Power of Attorney (Financial) | Power of Attorney |
| Power of Attorney (Medical) | Power of Attorney |
| Power of Attorney (HIPAA) | Power of Attorney |
| Advanced Medical Directive | Advanced Medical Directive |
| Last Will and Testament | Will |
| Individual Trust No-Tax Planning | Trust - No Tax Planning |
| Joint Trust No-Tax Planning | Trust - No Tax Planning |
| Affidavit | Affidavit |
| Deed | Deed |

> All three POA types (Financial, Medical, HIPAA) share the same portal service type
> "Power of Attorney" — each is still a separate submission with its own fee and comment.

## Step 1 — Load the Claim Doc

Scott will provide the MetLife claim `.docx` (generated by the `metlife-claim` skill).

Extract the **Portal Submission Table** from the doc — it contains one row per submission:

| # | Document | Service Type | Fee | Comment | Complete |
|---|----------|-------------|-----|---------|----------|

This table is the submission queue. Work through it top to bottom.

If Scott has not provided the claim doc, ask for it before opening the browser. Do not
attempt to reconstruct the line items from memory.

---

## Step 2 — Navigate to the Portal

1. Navigate to `https://attorneys.legalplans.com/my-cases`
2. Search for the client by member name
3. Keep this search results page open — you will return to it between submissions

---

## Step 3 — Submit Each Line Item (Loop)

For **each row** in the Portal Submission Table, repeat the following:

### 3a — Find the correct case row

In the search results table, find the row matching:
- **Service type** from the current line item
- **Status: "In Progress"**

**Before clicking**, confirm: "Found: [Client Name] — [Service Type] — In Progress"

**Edge cases:**
- **Multiple "In Progress" rows for the same service type** — show Scott the Service IDs and ask which to use
- **Status is "Final Claim Submitted"** — notify Scott and skip this line item; continue to the next
- **No matching row** — report what's visible and ask Scott how to proceed; do not skip silently

### 3b — Open the claim form

1. Click the row to expand it
2. Scroll down and click **"Submit Claim"**
3. Confirm the URL contains `/submit-claim` before filling

### 3c — Fill the form

| Field | Value |
|-------|-------|
| Date work started | Pre-filled — verify it matches the claim doc; update if empty or wrong |
| Date work finished | From claim doc |
| Is this case complete? | Always "Yes" |
| Telephone and email advice | $0.00 |
| Office advice | $0.00 |
| Standard fee | Line item fee from claim doc |
| Other / Extra fees | $0.00 |
| Comments | Comment from claim doc |

### 3d — Confirm before submitting

Show Scott a one-line confirmation before each submit:

```
[#/total] [Document name] — $[fee] — Ready to submit? (yes / yes to all / skip / stop)
```

**Response options:**
- **"yes"** — submit this one, then confirm again for the next
- **"yes to all"** — submit all remaining without further confirmation (Scott has reviewed the full list)
- **"skip"** — skip this line item, move to the next
- **"stop"** — halt the entire run

### 3e — Submit and log

1. Click **"Submit Claim"**
2. Note the confirmation number or status shown
3. Mark the row complete in your running log
4. Return to the search results page for the next item

## Important Rules

- **Never submit a line item without confirmation** unless Scott has said "yes to all"
- **Never guess the service type** — use the mapping table above; ask if uncertain
- **Always verify client name + service type + "In Progress" before expanding a row**
- **If the portal behaves unexpectedly** (login prompt, timeout, unexpected page) — stop the run and report; do not attempt workarounds
- **The claim doc is authoritative** — fees and comments come from the doc, not from memory or the fee schedule
